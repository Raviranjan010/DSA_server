## Section A: Concept-Based (Beginner)

Q1. What is the Base Case in a recursive function?
A) The longest path in the recursion tree
B) The condition under which the recursion stops
C) The initial arguments passed to the function
D) The mathematical base (e.g., base 10)
✅ Answer: [B]
📝 Explanation: A base case is a specific condition where the recursive function returns a direct value without making any further recursive calls, preventing infinite recursion.
🏢 Asked by: [General]

Q2. What data structure is used implicitly by the compiler/interpreter to manage recursive function calls?
A) Queue
B) Array
C) Stack
D) Linked List
✅ Answer: [C]
📝 Explanation: Function calls are pushed onto the Call Stack. When a function returns, its frame is popped from the Call Stack. Thus, heavy recursion can lead to Stack Overflow.
🏢 Asked by: [TCS]

Q3. What is Tail Recursion?
A) A recursive function where the recursive call is the very first statement.
B) A recursive function that only processes the tail (end) of an array.
C) A recursive function where the recursive call is the last executed statement, and no operations are done after it returns.
D) A recursion that splits into two equal halves.
✅ Answer: [C]
📝 Explanation: In tail recursion, the recursive call is the final action. Many modern compilers can optimize tail recursion (Tail Call Optimization) to prevent additional memory overhead on the Call Stack.
🏢 Asked by: [Amazon]

Q4. Which of the following problems naturally requires recursion or backtracking?
A) Finding the average of an array
B) Finding all subsets of a set
C) Searching for an element in an unsorted array
D) Finding the maximum in an array
✅ Answer: [B]
📝 Explanation: Generating combinations, permutations, subsets, etc., usually entails making multiple choices at each step, making recursion/backtracking the most natural approach.
🏢 Asked by: [Microsoft]

Q5. How does Backtracking differ from plain Recursion?
A) Backtracking is iterative instead of recursive
B) Backtracking prunes the recursion tree by undoing choices if they lead to an invalid state.
C) Backtracking doesn't use the call stack
D) Backtracking can only move backwards in an array
✅ Answer: [B]
📝 Explanation: Backtracking tries to build a solution incrementally, but "abandons" a path (backtracks) as soon as it determines that this path cannot possibly lead to a valid solution.
🏢 Asked by: [Adobe]

## Section B: Code Output / Tracing (Medium)

Q6. What will be the output of the following code?
```cpp
#include <iostream>
using namespace std;

void func(int n) {
    if (n <= 0) return;
    cout << n << " ";
    func(n - 1);
    cout << n << " ";
}

int main() {
    func(3);
    return 0;
}
```
A) 3 2 1
B) 1 2 3
C) 3 2 1 1 2 3
D) 3 2 1 0 1 2 3
✅ Answer: [C]
📝 Explanation: `func` prints the number before calling itself, and then prints it again after the recursion unwinds. Trace:
func(3): prints 3 -> calls func(2) -> [waits] -> prints 3
func(2): prints 2 -> calls func(1) -> [waits] -> prints 2
func(1): prints 1 -> calls func(0) -> [waits] -> prints 1
Output: 3 2 1 1 2 3
🏢 Asked by: [Infosys]

Q7. What is the output?
```cpp
int power(int a, int b) {
    if (b == 0) return 1;
    if (b % 2 == 0) return power(a, b / 2) * power(a, b / 2);
    return a * power(a, b - 1);
}
```
If we call `power(2, 4)`, what is returned?
A) 16
B) 8
C) 4
D) 2
✅ Answer: [A]
📝 Explanation: `power(2, 4)`: b is even -> returns power(2, 2) * power(2, 2)
`power(2, 2)`: b is even -> returns power(2, 1) * power(2, 1)
`power(2, 1)`: returns 2 * power(2, 0) = 2 * 1 = 2
So, `power(2, 2)` = 2 * 2 = 4
So, `power(2, 4)` = 4 * 4 = 16
🏢 Asked by: [Wipro]

Q8. Why is the naive recursive Fibonacci implementation `fib(n) = fib(n-1) + fib(n-2)` inefficient?
A) It uses too much heap memory
B) It recomputes the same subproblems repeatedly
C) It has O(N) time complexity
D) It cannot compute `fib(0)`
✅ Answer: [B]
📝 Explanation: Naive Fibonacci is inefficient because it calculates the same Fibonacci numbers over and over, leading to an exponential Time Complexity of O(2^N).

## Section C: Application & Problem Solving (Hard)

Q9. In the N-Queens problem, how do you efficiently check if a diagonal is under attack without looping through it?
A) We mathematically cannot, we must iterate.
B) Use separate boolean arrays for hashing. Left diagonals share same (row - col) and Right diagonals share same (row + col).
C) Check only adjacent cells.
D) Hash the board matrix into a string.
✅ Answer: [B]
📝 Explanation: Instead of an O(N) loop to check diagonal safety, we can use an O(1) hash map approach:
1. Left diagonal formula: `row - col` (adding an offset `N-1` prevents negative indices).
2. Right diagonal formula: `row + col`.
🏢 Asked by: [Meta]

Q10. For generating all valid parentheses combinations of `N` pairs, what condition must be true to place a closing parenthesis `)`?
A) The number of `)` placed so far must be strictily less than the number of `(`.
B) The number of `)` placed must equal `(`.
C) We can place `)` at any time as long as total `)` is less than `N`.
D) `(` should always be placed first.
✅ Answer: [A]
📝 Explanation: In generation, `)` can only be added if there is an unmatched `(` available, meaning count(`)`) < count(`(`).
🏢 Asked by: [Amazon]

Q11. Running Backtracking to find all permutations of an array/string of length N takes Time Complexity of:
A) O(N)
B) O(N * log N)
C) O(2^N)
D) O(N! * N)
✅ Answer: [D]
📝 Explanation: There are N! permutations. For each permutation, we take O(N) time to print/store it. Total time is O(N! * N).

## Section D: Competitive Programming Traps

Q12. What does deep recursion in C++ usually lead to if no optimization happens?
A) Memory Leak
B) Segmentation Fault (Stack Overflow)
C) Infinite loop
D) Compile-time error
✅ Answer: [B]
📝 Explanation: The Call Stack is limited (typically 8MB on Linux). Depth ~10^4 or 10^5 usually causes a Stack Overflow leading to a segmentation fault.
🏢 Asked by: [Codeforces]

Q13. You are generating subsets by passing the current choice array by VALUE `void solve(vector<int> curr)`. Why is this problematic in CP?
A) It won't compile
B) It causes an O(N) time copy for every recursive call, severely degrading performance.
C) The array `curr` loses its values between returns.
D) Vectors cannot be passed by value in recursion.
✅ Answer: [B]
📝 Explanation: Passing by value means the whole vector is deep-copied at each recursive step! Always pass by REFERENCE `void solve(vector<int>& curr)` and push/pop elements manually to backtrack.
🏢 Asked by: [Flipkart]
