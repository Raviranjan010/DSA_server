## Section A: Concept-Based (Beginner)

Q1. Which sorting algorithm is known for its "divide and conquer" approach and uses a pivot?
A) Merge Sort
B) Bubble Sort
C) Quick Sort
D) Insertion Sort
✅ Answer: [C]
📝 Explanation: Quick Sort picks an element as a pivot and partitions the given array around the picked pivot.
🏢 Asked by: [General]

Q2. What is the time complexity of a standard Binary Search?
A) O(N)
B) O(N log N)
C) O(log N)
D) O(1)
✅ Answer: [C]
📝 Explanation: In each step, Binary Search eliminates half of the remaining elements, leading to logarithmic time complexity O(log_2 N).
🏢 Asked by: [General]

Q3. What prerequisite is necessary for Binary Search to work?
A) The array must have no duplicates
B) The array must be sorted
C) The array must contain only positive integers
D) The array must be rotated
✅ Answer: [B]
📝 Explanation: Binary search relies on the sorted order of elements to definitively eliminate half the search space at each comparison.
🏢 Asked by: [TCS]

Q4. Which sorting algorithm's worst-case time complexity is O(N log N)?
A) Bubble Sort
B) Quick Sort
C) Merge Sort
D) Selection Sort
✅ Answer: [C]
📝 Explanation: Merge sort consistently divides the array in half and merges uniformly, giving a strict O(N log N) in best, average, and worst cases. Quick sort's worst case is O(N^2).
🏢 Asked by: [Infosys]

Q5. Which sorting algorithm is generally considered stable?
A) Quick Sort
B) Heap Sort
C) Merge Sort
D) Selection Sort
✅ Answer: [C]
📝 Explanation: Merge Sort preserves the relative order of equal elements during the merge phase, making it a stable sort.
🏢 Asked by: [Microsoft]

## Section B: Code Output / Tracing (Medium)

Q6. In the first pass of Bubble Sort on array `[5, 1, 4, 2, 8]`, what will the array look like?
A) `[1, 4, 2, 5, 8]`
B) `[1, 5, 4, 2, 8]`
C) `[5, 1, 4, 2, 8]`
D) `[1, 2, 4, 5, 8]`
✅ Answer: [A]
📝 Explanation: Trace:
5>1: swap -> [1, 5, 4, 2, 8]
5>4: swap -> [1, 4, 5, 2, 8]
5>2: swap -> [1, 4, 2, 5, 8]
5<8: no swap -> [1, 4, 2, 5, 8]
The heaviest element (8) is already at the end. The final state after pass 1 is `[1, 4, 2, 5, 8]`.
🏢 Asked by: [Wipro]

Q7. In a standard mid calculation `mid = (left + right) / 2;`, what bug can happen for very large arrays?
A) Precision loss
B) Integer Overflow
C) Null pointer exception
D) Division by zero
✅ Answer: [B]
📝 Explanation: If `left` and `right` are large, `left + right` can exceed `INT_MAX`, causing negative overflow. The safe formula is `mid = left + (right - left) / 2;`.
🏢 Asked by: [Google]

Q8. What happens in Quick Sort if the array is already sorted and we pick the first element as pivot?
A) It performs in O(N log N)
B) It triggers an infinite loop
C) It falls to O(N^2) worst-case time complexity
D) It sorts it in O(N) time
✅ Answer: [C]
📝 Explanation: If the first (or last) element is chosen as pivot on a sorted array, one partition will always have N-1 elements and the other 0 elements, completely defeating the "divide in half" advantage.
🏢 Asked by: [Amazon]

## Section C: Application & Problem Solving (Hard)

Q9. How do you find the first occurrence of a target element in a sorted array containing duplicates using Binary Search?
A) Stop immediately when target is found.
B) When `arr[mid] == target`, reduce `right` to `mid - 1` and keep searching.
C) When `arr[mid] == target`, increase `left` to `mid + 1` and keep searching.
D) Use a Hash Map instead.
✅ Answer: [B]
📝 Explanation: To find the *first* occurrence, finding the target is not enough. We must see if there's an even earlier instance by shrinking the right boundary (`right = mid - 1`) and storing the potential answer.
🏢 Asked by: [Meta]

Q10. Which algorithm would you use to solve "K-th largest element" optimally without sorting the entire array first?
A) Merge Sort
B) Quick Select
C) Insertion Sort
D) Linear Search
✅ Answer: [B]
📝 Explanation: Quick Select uses Quick Sort's partitioning scheme but only recursively calls on the half that contains the K-th element, yielding average O(N) time.
🏢 Asked by: [Microsoft]

Q11. You must sort a massive dataset of 10GB stored on disk using only 1GB of RAM. What algorithm do you use?
A) External Merge Sort
B) Quick Sort
C) Standard Merge Sort
D) Heap Sort
✅ Answer: [A]
📝 Explanation: External Merge Sort reads 1GB chunks, sorts them, writes to disk, and then merges the sorted chunks sequentially.
🏢 Asked by: [Amazon]

## Section D: Competitive Programming Traps

Q12. In C++, `std::sort` uses which algorithm under the hood?
A) Plain Quick Sort
B) IntroSort
C) Merge Sort
D) Timsort
✅ Answer: [B]
📝 Explanation: C++ `std::sort` uses IntroSort, which is a hybrid algorithm. It starts with Quick Sort, but switches to Heap Sort if the recursion depth gets too large (preventing O(N^2) worst-case), and uses Insertion Sort for very small partitions.
🏢 Asked by: [Codeforces]

Q13. In CP, if you need a STABLE sort, which standard C++ function should you use?
A) `std::sort`
B) `std::qsort`
C) `std::stable_sort`
D) `std::merge`
✅ Answer: [C]
📝 Explanation: `std::stable_sort` preserves the relative order of elements with equivalent keys (guarantees O(N log^2 N), or O(N log N) if extra memory is available).
🏢 Asked by: [Flipkart]
