# 🗺️ ARRAY MASTERY ROADMAP

> "From Zero to FAANG-Ready in 7 Steps"

If you are reading this for the first time, follow this exact order to master Arrays. **Do not skip steps.**

---

### 🟢 Phase 1: The Foundations (Day 1)
**File**: `01_arrays_foundations.md`
- Understand how memory works (Contiguous allocation).
- Learn why arrays are $O(1)$ access (The Formula).
- Understand the difference between Stack (Static) and Heap (Vector) memory.
- **Goal**: Be able to explain *why* arrays are fast and how they are stored.

### 🟡 Phase 2: Basic Operations & Sorting (Day 2)
**File**: `01_arrays_foundations.md` (Chapters 5 & 6)
- Master Traversal, Insertion, and Deletion.
- Learn Linear vs Binary Search.
- Understand the 5 main sorting algorithms (Bubble, Selection, Insertion, Merge, Quick).
- **Goal**: Be able to write search and sort from scratch.

### 🟠 Phase 3: The Pattern Bible - Part A (Day 3-4)
**File**: `02_arrays_patterns.md` (Patterns 1–5)
- **Prefix Sum**: Range sum queries.
- **Sliding Window**: Subarray problems.
- **Two Pointers**: Sorted array problems.
- **Kadane's Algorithm**: Maximum sum subarray.
- **Binary Search Variants**: First/Last occurrence, Rotated search.
- **Goal**: Recognize these patterns in problem statements.

### 🔴 Phase 4: The Pattern Bible - Part B (Day 5-6)
**File**: `02_arrays_patterns.md` (Patterns 6–12)
- **Dutch National Flag**: 0, 1, 2 sorting.
- **Cyclic Sort**: Problems with range [1, n].
- **XOR Tricks**: Single number, duplicate problems.
- **2D Matrix**: Rotation and Spiral Traversal.
- **Difference Array**: Range updates.
- **Merge Intervals**: Overlapping ranges.

### 🟣 Phase 5: Shortcuts & Industry Tricks (Day 7)
**File**: `03_arrays_shortcuts.md`
- Memorize the "Pattern Recognition Bible".
- Learn the C++ STL one-liners (sort, unique, lower_bound).
- Set up your Competitive Programming template.

### 🔵 Phase 6: The "Must-Solve" Problems (Day 8-10)
**File**: `04_arrays_company_questions.md`
- Solve the **Top 10 Beginner Problems** first.
- Move to **Easy** → **Medium** → **Hard** levels.
- Read the "Approach" and "Dry Run" for every problem you solve.

### ⚪ Phase 7: Final Assessment (Day 11)
**File**: `05_arrays_mcq_bank.md`
- Take the 80-question MCQ test.
- If you score >90%, you have mastered Arrays.

---

## 💡 Pro-Tips for Mastery
1. **Paper First**: Never code before dry-running on paper.
2. **Visualize**: Draw the pointers (left, right, mid) moving.
3. **Complexity**: Always ask "Can I do this in $O(n)$?" before settling for $O(n^2)$.
4. **STL is Your Friend**: Master `std::vector` and `<algorithm>` header.
