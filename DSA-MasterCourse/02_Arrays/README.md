## 🎯 Pattern Map Table

| Pattern | When to Use | Time Complexity | Key Problems |
|---------|-------------|-----------------|--------------|
| **Two Pointer** | Sorted arrays, pairs, palindromes, partitioning | O(n) | Two Sum, Container With Most Water |
| **Sliding Window** | Subarrays, substrings, consecutive elements | O(n) | Maximum Sum Subarray of Size K |
| **Prefix Sum** | Range sum queries, subarray counting | O(1) per query | Subarray Sum Equals K |
| **Kadane's Algorithm** | Maximum/minimum subarray sum/product | O(n) | Maximum Subarray |
| **Binary Search** | Sorted data, optimization problems | O(log n) | Find Peak Element, Search in Rotated Array |
| **Hashing + Arrays** | Frequency counting, two-sum variants | O(n) time, O(n) space | Contains Duplicate, Group Anagrams |
| **Sorting + Arrays** | When order matters, duplicate removal | O(n log n) | Merge Intervals, 3Sum |

## 🔑 Quick Reference: Pattern Decision Tree

```
Problem mentions "sorted array"?
├─ YES → Two Pointer or Binary Search
│  ├─ Looking for pairs? → Two Pointer
│  └─ Finding element/optimizing? → Binary Search
│
└─ NO → Check other patterns:
   ├─ "Subarray" or "consecutive"? → Sliding Window
   ├─ "Range sum" or "count subarrays"? → Prefix Sum
   ├─ "Maximum subarray sum"? → Kadane's Algorithm
   ├─ "Frequency" or "count occurrences"? → Hashing
   └─ "All pairs" or "combinations"? → Sorting + Two Pointer
```

## 💡 Problem-Solving Strategy

```
Step 1: Read problem carefully (2 min)
   - Underline key words
   - Identify constraints
   - Note edge cases

Step 2: Identify the pattern (3 min)
   - Use Pattern Decision Tree
   - Match keywords to patterns
   - Think of similar problems

Step 3: Brute force approach (5 min)
   - What's the naive solution?
   - What's its complexity?
   - Can you code it quickly?

Step 4: Optimize (10 min)
   - What information is redundant?
   - Can you use a pattern?
   - What's the target complexity?

Step 5: Write code (10 min)
   - Start with template
   - Add comments
   - Handle edge cases

Step 6: Test (5 min)
   - Normal cases
   - Edge cases (empty, single element)
   - Large inputs

Step 7: Analyze complexity (5 min)
   - Time complexity
   - Space complexity
   - Can it be improved?
```

## ⚠️ Common Pitfalls to Avoid

1. **Off-by-one errors** — Always verify loop bounds
2. **Empty array** — Check `if(n == 0)` first
3. **Single element** — Check `if(n == 1)` edge case
4. **Integer overflow** — Use `long long` for sums
5. **Negative indices** — Validate before accessing
6. **Modifying while iterating** — Creates bugs
7. **Forgetting to sort** — Many patterns require sorted input
