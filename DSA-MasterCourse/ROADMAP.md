## 📅 Milestone Checkpoints

### Checkpoint 1: After Phase 1 (Week 4)
**You should be able to:**
- Write basic C++ programs using STL
- Analyze time/space complexity of any code
- Identify bottlenecks in algorithms

**Self-Test:**
- Solve 10 Easy array problems on LeetCode
- Explain Big-O to a friend in simple terms

---

### Checkpoint 2: After Phase 2 (Week 12)
**You should be able to:**
- Implement all linear data structures from scratch
- Recognize sliding window, two pointer patterns
- Solve medium array/string problems independently

**Self-Test:**
- Complete LeetCode Top Interview 50 (Easy/Medium)
- Score 80%+ on all MCQs from topics 02-09

---

### Checkpoint 3: After Phase 3 (Week 18)
**You should be able to:**
- Traverse and manipulate trees/graphs
- Implement BFS/DFS without reference
- Solve tree/graph medium problems

**Self-Test:**
- Solve 20 tree problems + 20 graph problems
- Implement Dijkstra's algorithm from memory

---

### Checkpoint 4: After Phase 4 (Week 22)
**You should be able to:**
- Identify when to use DP vs Greedy
- Write recursive + memoized + tabulated DP solutions
- Solve classic DP problems (knapsack, LCS, LIS)

**Self-Test:**
- Solve 30 DP problems (10 Easy, 15 Medium, 5 Hard)
- Explain DP approach aloud for any problem

---

### Checkpoint 5: After Phase 5 (Week 25)
**You should be able to:**
- Implement advanced graph algorithms
- Solve hard DP variants (tree DP, bitmask DP)
- Handle number theory and combinatorics problems

**Self-Test:**
- Participate in 2 Codeforces contests
- Score 70%+ on advanced MCQs

---

### Checkpoint 6: After Phase 6 (Week 26+)
**You should be able to:**
- Solve any interview problem in 30-40 minutes
- Explain your approach clearly under pressure
- Handle follow-up questions confidently

**Self-Test:**
- Complete 5 mock interviews (friends/platforms)
- Solve company-specific question sets
- Get comfortable with whiteboard coding

## 🎯 Fast-Track Alternative (3 Months)

If you're short on time, follow this accelerated path:

```
Week 1-2:  00 → 01 → 02 → 03 (Foundation + Arrays/Strings)
Week 3-4:  05 → 06 → 07 → 09 (Sorting, LL, Stack, Hashing)
Week 5-6:  10 → 11 → 12 → 14 (Trees, BST, Heaps, Graphs)
Week 7-8:  15 → 20 → 23 (DP, Advanced Graphs, Company Qs)
Week 9-10: 13 → 16 → 18 → 21 (Tries, Greedy, Bits, Adv DP)
Week 11-12: Mock interviews + Revision + Cheatsheets
```

**What you skip (study later):**
- 04 (Recursion) — Learn alongside DP
- 08 (Queue) — Less frequently asked
- 17 (Divide & Conquer) — Covered in sorting
- 19 (Segment Tree) — Rare in interviews
- 22 (CP Extras) — Only for competitive programming

## 📊 Time Allocation Guide

### Per Topic (Average 1.5 weeks):
```
Theory (Notes):      30% (3-4 hours)
  • Read notes thoroughly
  • Study diagrams
  • Understand dry runs

Code Practice:       40% (5-6 hours)
  • Type code from scratch
  • Run test cases
  • Experiment with variations

Problem Solving:     30% (4-5 hours)
  • Solve 10-15 problems
  • Start Easy → Medium → Hard
  • Time yourself (30 min/problem)
```

### Weekly Breakdown (6-month plan):
```
Monday-Friday:   2-3 hours/day
  • 1 hour: Theory/Notes
  • 1-2 hours: Problem solving

Saturday:        4-5 hours
  • Revise week's topics
  • Solve additional problems
  • Take MCQ tests

Sunday:          2 hours (light)
  • Plan next week
  • Watch related videos
  • Join coding communities
```
