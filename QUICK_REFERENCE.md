## C++ QUICK REFERENCE

```
MUST KNOW CONTAINERS:
┌──────────────────────────────────────────┐
│ vector<int> v;           │ Dynamic array  │
│ v.push_back(5);          │ O(1) amortized │
│ v[0];                    │ O(1) access    │
├──────────────────────────────────────────┤
│ unordered_map<int, int>  │ Hash map       │
│ mp[key] = value;         │ O(1) average   │
│ mp.count(key);           │ Check exists   │
├──────────────────────────────────────────┤
│ set<int> s;              │ Ordered set    │
│ s.insert(5);             │ O(log n)       │
│ s.find(5);               │ Search O(log n)│
├──────────────────────────────────────────┤
│ queue<int> q;            │ FIFO           │
│ q.push(5); q.pop();      │ O(1) each      │
│ q.front();               │ O(1) peek      │
├──────────────────────────────────────────┤
│ stack<int> st;           │ LIFO           │
│ st.push(5); st.pop();    │ O(1) each      │
│ st.top();                │ O(1) peek      │
└──────────────────────────────────────────┘

ALGORITHM FUNCTIONS:
sort(v.begin(), v.end())           │ O(n log n)
reverse(v.begin(), v.end())        │ O(n)
find(v.begin(), v.end(), val)      │ O(n)
binary_search(v.begin(), v.end())  │ O(log n)
max_element(v.begin(), v.end())    │ O(n)
```

## TOP 10 MISTAKES (Avoid These!)

```
1. Copying solutions immediately
   → Wait 20 min before checking

2. Not doing spaced repetition
   → Schedule reviews (use Guide 3)

3. Solving too many problems shallow
   → 200 problems deep > 500 surface level

4. Not timing yourself
   → Speed matters for interviews

5. Skipping weak topics
   → Weak = highest ROI
   → Fix weak, overall improves

6. Giving up Day 15-30
   → This is when learning actually happens
   → Push through!

7. Not explaining code out loud
   → Interview requires communication
   → Practice explaining

8. Memorizing solutions
   → Learn patterns instead
   → Understand "why", not "what"

9. Only solving LeetCode
   → Mix LeetCode + company-specific
   → Mix easy + medium + hard

10. Not tracking progress
    → You can't improve what you don't measure
    → Keep spreadsheet
```
